About

Explanation:

This project was the result of a question that came up on my blog post about programmaticly resetting the history of a navigation stack for an individual tab.

Normal Tab:

The Movies tab will maintain the normal navigation flow, so when a user navigates from the first page to the second page, leaving and re-entering the Movies tab will not reset the nav stack. The user will re-enter at whichever of those pages they last saw activated.

Reset Tab:

The Bands tab will always reset back to the root of the nav stack - that being the band listing. In order to do this, I handled the ionSelect event on the Bands tab, and analyzed its internal _views property. That functionality can probably be extended to build some deeper navigational programming. You can find the code that provides that functionality in app/pages/tabs/tabs.ts.

To test:

  • Tap on the Bands tab
  • Tap on any band to view a list of that band's songs
  • Tap on the Movies tab
  • Tap on any movie to see a song or two from its soundtrack
  • Tap on the Bands tab again
  • You should be brought back to the list of bands
  • Tap on the Movies tab again
  • You should be brought back to the list of songs from the movie you previously selected
