About

Explanation:

This project was the result of a question that came up on my blog post about programmaticly resetting the history of a navigation stack for an individual tab.

Normal Tab:

The Europe tab will maintain the normal navigation flow, so when a user navigates from the first page to the second page, leaving and re-entering the Europe tab will not reset the nav stack. The user will re-enter at whichever of those pages they last saw activated.

Reset Tab:

The USA tab will always reset back to the root of the nav stack - that being the state listing. In order to do this, I handled the ionSelect event on the USA tab, and analyzed its internal _views property. That functionality can probably be extended to build some deeper navigational programming. You can find the code that provides that functionality in app/pages/tabs/tabs.ts.

To test:

  • Tap on the USA tab
  • Tap on any state to view some cities within that state
  • Tap on the Europe tab
  • Tap on any country to see some cities within that country
  • Tap on the USA tab again
  • You should be brought back to the list of states
  • Tap on the Europe tab again
  • You should be brought back to the list of cities from the country you previously selected
